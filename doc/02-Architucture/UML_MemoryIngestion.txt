'──────────────────────────────────────────────────────────────
'  RELATIONSHIPS / FLOW
'──────────────────────────────────────────────────────────────

MemoryManager "1" o-- "0..*" MemoryRecord : owns live records

MemoryIngestionManager --> MemoryManager : reads records
MemoryIngestionManager --> MemoryChunker : builds entries
MemoryIngestionManager --> MemoryVectorStore : writes/replaces vectors

MemoryChunker --> MemoryRecord : reads Q/A + metadata
MemoryChunker ..> MemoryVectorEntry : creates entry payloads

MemoryVectorStore --> MemoryChromaDB : persists vectors
MemoryVectorStore ..> MemoryVectorEntry : stores entries

MemoryVectorEntry --> MemoryRecord : metadata.record_id links to parent

'──────────────────────────────────────────────────────────────
'  NOTES
'──────────────────────────────────────────────────────────────

note right of MemoryManager
  Existing Memory Recording object.

  It remains the owner of:
  - full Q/A truth
  - live MemoryRecord list
  - persistence state

  It does not perform vector ingestion.
end note

note right of MemoryIngestionManager
  Orchestrates ingestion only.

  Main flow:
  record_id
  -> find MemoryRecord
  -> build vector entries
  -> replace record vectors

  May run async after durable save.
end note

note bottom of MemoryChunker
  Creates deterministic/extractive entries:
  - record_handle
  - question blocks
  - answer blocks

  No LLM summary.
  No inferred intent.
end note

note bottom of MemoryVectorStore
  Owns dedicated memory ChromaDB.

  It does not own MemoryRecord truth.
  It does not perform retrieval ranking.
  It does not compress context.
end note

note bottom of MemoryVectorEntry
  Chroma entry has its own id.

  Parent link is stored in metadata:
  record_id -> MemoryRecord.record_id
end note
