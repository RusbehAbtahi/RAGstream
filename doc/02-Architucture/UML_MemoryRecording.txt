'──────────────────────────────────────────────────────────────
'  RELATIONSHIPS / FLOW
'──────────────────────────────────────────────────────────────

StreamlitUI --> MemoryActions : button calls
StreamlitUI --> AppController : reads project context

MemoryActions --> MemoryManager : sync + capture

MemoryManager "1" *-- "0..*" MemoryRecord : owns records

MemoryManager --> RagMemFile : append .ragmem
MemoryManager --> RagMetaJson : rewrite .ragmeta.json
MemoryManager --> MemorySQLite : refresh active file_id

MemoryRecord --> MemoryRecord : parent_id
MemoryManager ..> MemoryRecord : creates

MemoryRecord ..> RagMemFile : to_ragmem_block()
MemoryRecord ..> MemorySQLite : to_index_dict()

'──────────────────────────────────────────────────────────────
'  NOTES
'──────────────────────────────────────────────────────────────

note right of StreamlitUI
  GUI is view/editor only.

  It provides:
  - raw prompt text
  - manual response text
  - current tag/user-keyword edits
  - active project name
  - embedded files snapshot

  It does not own memory truth.
end note

note right of MemoryActions
  Reusable capture workflow.

  Used by:
  - manual memory feed now
  - later LLM response capture
  - later tool result capture

  Keeps button callbacks thin.
end note

note right of MemoryManager
  Represents one active memory history.

  Owns:
  - file_id
  - filenames
  - records list
  - MetaInfo
  - persistence synchronization

  No MemorySession class.
end note

note bottom of MemoryRecord
  Atomic memory unit:
  one accepted input/output pair.

  input_text is raw Prompt,
  not SuperPrompt.
end note

note bottom of RagMetaJson
  .ragmeta.json is rebuilt from
  MemoryManager.records.

  This keeps GUI tag/user-keyword
  edits synchronized without
  per-widget persistence callbacks.
end note

note bottom of MemorySQLite
  SQLite refresh is scoped to
  the active file_id.

  Records are upserted by:
  file_id + record_id.
end note
